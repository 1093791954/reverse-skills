# 缺口滑块（Slider Distance）

## 1. 题型描述
- 给定背景图（含一个被涂色/挖空的缺口）和一张小滑块图，要求把小滑块从起点拖到缺口位置。
- 代表厂商：极验 v3 / 网易易盾 / 顶象 / 数美 / 腾讯防水墙普通版 / Vaptcha / 阿里 ali140 / DataDome 滑块。

## 2. 关键检测维度
- **滑动距离精确度**：±2~5px 的容忍。
- **轨迹形态**：dx,dy,t 序列；要求加速度变化、回弹、停顿等"人类特征"。
- **总耗时**：通常 1.0~3.0s，过快过慢都失败。
- **客户端伪造检测**：mouse event 是否来自真实用户（trusted 标志，`MouseEvent.isTrusted`）。
- **图片缩放对齐**：CSS 显示尺寸 vs 真实像素尺寸；需按比例换算缺口距离。

## 3. 缺口距离计算（合法授权研究）
方法对照：
| 方法 | 代价 | 准确率 |
|------|------|--------|
| ddddocr `slide_match(target, background)` | 极低 | 90~95% |
| OpenCV Canny + 模板匹配 (`cv2.matchTemplate(bg, target, cv2.TM_CCOEFF_NORMED)`) | 低 | 80~95% |
| Sobel/拉普拉斯找暗块 | 低 | 70~85% |
| YOLOv5/YOLOv8 训练专用模型 | 中 | 95%+ |
| Siamese / U-Net 预测缺口 mask | 中高 | 95%+ |
| LLM 多模态（GPT-4o / Claude Vision） | 高（API 费） | 95%+，但慢 |

## 4. 轨迹生成思路
- **三段式**（起步加速 → 中段平稳 → 减速）：最经典，多数中小厂商够用。
- **贝塞尔曲线**：4 个控制点，二次/三次贝塞尔生成 dx 序列，再二阶差分得 dt。
- **录制 + KDE 重采样**：录人类样本 100~500 条，按距离分桶，KDE 估计抽样。
- **GAN 轨迹生成**：训练 LSTM/GAN，输出像真人。
- **超调 + 回弹**：故意冲过缺口 5-10px，再回拉 ±2px，命中率显著提升。
- 注意：在 t 维度也要加抖动，纯固定 dt 会被识破。

## 6. 防御性分析步骤
1. 在合法授权环境抓 100+ 次成功提交，记录 (距离, 总时长, 轨迹长度, dx 分布, dy 抖动)。
2. 训练距离回归 / 检测模型；分别评估各个 CV 算法。
3. 对失败案例 dump 服务端响应码与 message，建立失败原因分类（距离差/速度异常/指纹/IP）。
4. 最后串通 → 端到端测通过率，与 baseline（人手做）对比。

## 7. 缓解 / 趋势
- 厂商加干扰条纹、阴影、彩色噪声让模板匹配失败。
- 双缺口/移动缺口（极验 v4 hard）。
- 真假缺口混淆（顶象推理）。
- 客户端 motion 强校验，纯模拟必失败。
