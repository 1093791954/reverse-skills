# 拼图滑块（slider-puzzle）

> 配套 SKILL.md Path 3。

## 1. 题型特征

- 一张完整图被打乱成 N×M 方块；用户拖动滑块/箭头按钮还原顺序
- 或：一块需要拖到正确位置才能完成图片
- 比"缺口滑块"复杂：需要先识别"目标排列"再求解最少步数

代表厂商：极验、数美、部分自研。

## 2. 检测维度

同缺口滑块 + "操作步数"维度（最优解 vs 用户实际步数）。

## 3. CV 路径

1. **特征匹配（SIFT / ORB）**：提取每个 tile 的特征，与原图局部匹配。
2. **YOLO 块定位**：标注每个 tile 在原图的 bbox。
3. **PSNR / SSIM 比对**：候选排列与"已知正确图"的相似度。
4. **CLIP 整图相似度**：如果题目是"还原成可识别物体"，可以用 CLIP 评估当前排列与文字描述的相似度（如"a photo of a cat"）。

## 4. 轨迹

每次只拖一个 tile / 按一次按钮，每个动作都符合"三段式 / 贝塞尔"基本要求。

## 5. 已知研究

- [NEEDS_VERIFICATION] 较缺口滑块少；CSDN / 知乎 偶有"拼图验证码"分析

## 6. 防御性分析思路

- 自有站点尽量避免使用纯拼图（用户体验差，CV 攻破并不难）。

## 7. 待研究问题

- 各厂商对"步数"的容忍度
- 是否记录"拖动取消"事件作为反作弊信号
