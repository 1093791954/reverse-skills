# Drafting Patterns

Use these patterns as starting points. Replace placeholders with facts and delete sections that do not fit.

## Style

- Prefer direct, precise Chinese with official tone.
- Keep each paragraph focused on one function: basis, fact, decision, measure, requirement, deadline, contact.
- Use active responsible units and clear deadlines where action is required.
- Avoid slogans, empty intensifiers, unsupported legal claims, and vague requirements such as "高度重视" without concrete action.
- Use `根据...`, `为...`, `经研究...`, `现就...通知如下` only when the logic is true.
- Keep terminology consistent with laws, policies, organization names, project names, and prior documents.

## Title Formulas

- General: `发文机关名称 + 关于 + 事由 + 的 + 文种`
- Forwarding/issuing: `发文机关名称 + 关于印发 + 文件名称 + 的通知`
- Meeting: `发文机关名称 + 关于召开 + 会议名称 + 的通知`
- Request: `发文机关名称 + 关于 + 请求事项 + 的请示`
- Report: `发文机关名称 + 关于 + 工作/情况 + 的报告`
- Letter: `发文机关名称 + 关于 + 商洽/征求/请求/答复事项 + 的函`
- Reply: `发文机关名称 + 关于 + 请示事项 + 的批复`

## 通知 Skeleton

```text
（主送机关）：

为（目的），根据（依据），经（研究/批准），现就有关事项通知如下。

一、（事项一）
（具体要求、责任单位、时限。）

二、（事项二）
（具体要求、责任单位、时限。）

三、有关要求
（执行、反馈、联系人、报送方式等。）

附件：1. （附件名称）
      2. （附件名称）

（发文机关署名）
XXXX年X月X日
```

## 请示 Skeleton

```text
（主送机关）：

为（目的/背景），根据（政策依据/上级要求），我单位拟（事项）。现将有关情况请示如下。

一、基本情况
（说明事实、必要性、已有工作。）

二、请示事项
（明确请求批准、指示或支持的事项。）

三、有关说明
（资金、人员、风险、协同、后续安排等。）

妥否，请批示。

（发文机关署名）
XXXX年X月X日
```

## 报告 Skeleton

```text
（主送机关）：

根据（依据/要求），现将（事项）有关情况报告如下。

一、基本情况
（事实和进展。）

二、主要做法和成效
（工作措施和结果。）

三、存在问题
（客观说明问题和原因。）

四、下一步工作
（后续安排。）

特此报告。

（发文机关署名）
XXXX年X月X日
```

## 函 Skeleton

```text
（主送机关）：

（说明来由、依据、背景。）为（目的），现就（事项）函告/函商如下。

一、（事项）
（具体内容。）

二、（请求或回复）
（需要对方办理、支持、确认或知悉的内容。）

妥否，请函复。/特此函复。/特此函达。

（发文机关署名）
XXXX年X月X日
```

## 批复 Skeleton

```text
（主送机关）：

你单位《（请示标题）》（文号）收悉。经研究，现批复如下。

一、（批复意见）
（同意/原则同意/不同意及理由。）

二、（执行要求）
（条件、程序、监督、期限等。）

此复。

（发文机关署名）
XXXX年X月X日
```

## 纪要 Skeleton

```text
（会议名称）纪要

XXXX年X月X日，（主持人/单位）主持召开（会议名称），研究（议题）。会议听取了（汇报），讨论了（事项），形成如下纪要。

一、会议认为
（共识、判断、原则。）

二、会议议定
（决定事项、责任单位、完成时限。）

三、会议要求
（落实、反馈、督办要求。）
```

## Common Edits

- Convert "拟请领导同意" to a proper请示 ending when seeking approval.
- Remove approval requests from报告; if approval is needed, change the文种 to请示.
- Replace vague verbs such as "做好相关工作" with responsible unit + action + deadline.
- Verify all organization names against official names; do not use casual abbreviations in titles or主送机关.
- Ensure every attachment named in正文 exists and every attached file is mentioned if formal routing requires it.
