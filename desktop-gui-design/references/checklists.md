# Desktop GUI Design Checklists

## Workspace

- Primary object and current context are always clear.
- Navigation, content, inspector/property panel, status, and command surfaces have distinct roles.
- Selection state is visible and unambiguous.

## Command Placement

- Frequent commands: toolbar or keyboard shortcut.
- Contextual commands: right-click/context menu or inline action.
- Full command coverage: menu bar or command palette.
- Dangerous commands: confirmation, undo, or delayed destructive execution.

## Density

- Dense views still preserve alignment, grouping, and scan paths.
- Tables/lists are used when comparison matters.
- Cards are used only when each item is a meaningful object summary.

## Settings

- Group by user task.
- Put risky settings near explanation and reset/recovery options.
- Avoid requiring restarts unless technically necessary.

## Accessibility

- Keyboard users can reach all primary commands.
- Focus state and selected state are visually distinct.
- Status and progress are communicated without color alone.
