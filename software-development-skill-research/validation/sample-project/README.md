# Validation Sample Project

This is a compact fixture used to validate the first software-development skills. It is not intended to build or run; each file represents a realistic review or planning artifact for one or more skills.

## Files

- `api/orders-openapi.yaml`: backend API contract, pagination, auth, errors.
- `db/001_add_orders.sql`: migration review fixture.
- `frontend/OrderFilter.tsx`: frontend state/accessibility fixture.
- `desktop/electron-ipc.ts`: desktop GUI IPC security fixture.
- `ops/orders-slo.yaml`: SRE production readiness and SLO fixture.
- `architecture/adr-001-order-boundary.md`: software architecture/ADR fixture.
