# 点选汉字 / 字母 / 图标（Click Character / Icon）

## 1. 题型描述
- 给一张图，按提示语顺序点击其中的汉字/字母/图标，按指定顺序提交点击坐标列表。
- 代表厂商：极验 v3 文字点选 / 极验 v4 icon-crush / 顶象点选 / 腾讯防水墙文字点选 / 数美点选 / Vaptcha 语义点选。
- 提示形式：
  - 顺序点选汉字：「请依次点选 风、云、雨」
  - 图标点选：「请点击 ↓ 中相同的图标」
  - 语义点选：「请点击图中的水果」

## 2. 关键检测维度
- 点击坐标精度（容忍约 ±10~15px）。
- 点击顺序必须正确。
- 点击时间间隔（人类两次点击 200~600ms，纯随机失败率高）。
- 鼠标 entry / hover 行为前置（瞬移到坐标点击容易被识破）。
- 提交参数加密（多为 AES/DES + base64）。

## 3. 解题思路（合法授权场景）
**两阶段**：
- **检测**：YOLOv5/v8 训练点选数据集，输出每个候选的 bbox。
- **匹配**：把提示语图（小图）与候选图（bbox 内截图）做 Siamese / ArcFace 特征向量比对。

变种：
- **OCR + 字典匹配**：DBNet/CRNN/ddddocr OCR 提示语 + 候选区文字，再按字符串匹配。
- **CLIP / OpenCLIP**：语义点选场景，prompt + image embedding 余弦相似度。
- **LLM 多模态**：GPT-4o / Claude Vision / Gemini 直接给提示语 + 图片返回点击坐标，准确率高但成本贵。
- **BLIP-2 / InstructBLIP**：开源多模态，不付费替代方案。

## 4. 已公开研究
- CSDN「文字点选验证码识别(上)-YOLO 位置识别」：抖音点选案例。
- CSDN「YOLOv5 文字点选验证码识别」：YOLOv5 + InsightFace Loss + Triplet loss + ONNX。
- CSDN「告别手动点选！用 ddddocr 1.3 搞定汉字验证码自动识别」：CNN + 改进 YOLO 混合架构。
- CSDN「从零构建文字点选验证码识别模型：YOLO 检测与特征匹配实战」：YOLO + ArcFace 三元组损失。
- CSDN「2026 点选验证码终极实战：OCR+语义匹配双路径」：DBNet++/改进 CRNN + YOLOv8s + MobileCLIP，55% → 95%+。
- CSDN「【实战指南】高效中文点选验证码识别方案解析」：YOLOv5/v8/SSD 对比、Focal Loss、NMS 后处理。
- CSDN「验证码智能破解实战：OCR+LLM 语义识别」：LLM 视觉协同。
- CSDN「顶象点选验证码」：图片数组生成与拼合。

## 5. 防御性分析步骤
1. 收集 1k~10k 张点选图 + 标注（合法授权环境，例如内部测试站）。
2. 训练 YOLOv8s/x 做候选检测；评估 mAP@0.5。
3. 训练 ArcFace 做配对（若同字体可只比 OCR 字符串）。
4. 端到端：candidates → match → 排序点击 → 加轨迹噪声。
5. 与基线（人工 + LLM 兜底）做 A/B。

## 6. 缓解 / 趋势
- 字体扭曲增强、彩色背景、干扰字符。
- 提示语换图（先 OCR 提示语图、再匹配候选）。
- 「顺序」从固定顺序改成"按笔画顺序"或"按拼音首字母"，需要中文知识图谱。
- 语义点选用很抽象的 prompt（"找出最重的"），CLIP 也吃力。

## 7. 待研究
- LLM 多模态（GPT-4o / Claude Vision）在中文古汉字点选的实际准确率。
- 数据增强 pipeline 在小样本（< 500）下的提升幅度。
- 提示语图与候选图同源 vs 异源对配对模型的影响。
