# Frontend UI Design

## Workflow

1. Identify the product type: SaaS/admin, content site, ecommerce, creative tool, data dashboard, or workflow app.
2. Start from the user task and information hierarchy, then choose layout, navigation, density, and component states.
3. Use the existing design system first. If none exists, define a small local system for spacing, type, color roles, controls, and states.
4. Design every meaningful state: default, hover, focus, active, disabled, loading, empty, error, success, selected, and destructive confirmation.
5. Check responsive behavior, accessibility, text fit, scanning path, and repeated-use ergonomics.

## Core Rules

- Operational tools should be dense, quiet, and efficient rather than hero-heavy or decorative.
- Forms need clear labels, helper text, validation, error recovery, and submit feedback.
- Dashboards should support scanning, comparison, filtering, and repeated action.
- Accessibility starts in design: contrast, focus order, hit targets, keyboard path, and error recovery.
