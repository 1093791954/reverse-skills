# Frontend UI Design Checklists

## Layout

- Make the primary task visible without requiring explanatory text.
- Use visual hierarchy for importance, not decoration.
- Keep repeated workflows compact and predictable.
- Reserve hero-scale type for actual hero contexts.

## Components

- Define default, hover, focus, active, disabled, loading, error, empty, success, and selected states.
- Ensure buttons and controls have stable dimensions.
- Use icons for familiar tool actions where appropriate.

## Forms

- Separate label, placeholder, helper text, validation, and default value.
- Put errors near the control and summarize only when useful.
- Preserve user input after failed submit.

## Dashboards

- Optimize for scanning, sorting, filtering, comparison, and drill-down.
- Avoid card-in-card layouts.
- Use tables or dense lists when comparison matters.

## Accessibility

- Meet contrast and focus visibility requirements.
- Ensure keyboard users can complete the same workflow.
- Do not rely on color alone for status.
