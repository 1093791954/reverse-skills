# 鼠标 / 触摸轨迹生成（贝塞尔 / 录制重放 / GAN）

## 1. 检测维度
- **dx 分布**：人类滑块 dx 多为 0~3 像素增量（高频低值），极少出现常数差。
- **dy 抖动**：y 方向 ±1~3 px 抖动，纯零行（直线）必失败。
- **dt 抖动**：相邻事件间隔 4~25 ms，浮动有抖动。
- **总耗时**：滑块 1.0~3.5s 最常见；过短 / 过长（< 600ms 或 > 6s）异常。
- **加速度曲线**：起步加速 → 平稳 → 减速 → 微调。
- **回弹 / 过冲**：80% 真人会过冲再回拉。
- **isTrusted 标志**：浏览器原生事件 `isTrusted=true`，dispatchEvent 模拟时为 false（部分厂商必查）。
- **PointerEvent 压力**：触屏 pressure 0.0~1.0，PC 默认 0；移动端必填。

## 2. 生成方法
| 方法 | 复杂度 | 通过率 |
|------|-------|--------|
| 三段加减速（Easing） | 低 | 中 |
| 二/三阶贝塞尔曲线 | 低 | 中高 |
| 真人录制回放（按距离分桶） | 中 | 高 |
| 录制 + KDE 重采样 | 中 | 高 |
| LSTM/Transformer 自回归 | 高 | 高 |
| GAN（mouseGAN/CursorGAN） | 高 | 高 |
| WGAN-GP（dt 分布） | 高 | 高 |

## 3. 实操要点
- 除非对方真把 `isTrusted` 当强校验，CDP `Input.dispatchMouseEvent` 即可（事件来自 Chrome 内核，标志 true）。
- 滚动也要加：mousedown 之前 200~500ms 有页面 mousemove。
- 触摸轨迹 + pressure：PC UA 不要假装触屏（不一致直接拒）。
- 提交 trace 时通常做 base64 + 简单变换，要按厂商加密包装。

## 5. 防御性分析步骤
1. 录制真实样本：在合法授权环境下录 200+ 条 (距离, dx 列表, dy 列表, dt 列表, 总耗时)。
2. 分桶：按距离 `[0,50)`、`[50,100)`、... 分，每桶画 KDE。
3. 生成器：对目标距离从对应桶里 KDE 采样 + 加随机抖动。
4. 评估：看服务端拒绝率与 dx/dy/dt 的统计特征差异。
5. 反复迭代：拒绝集合做 hard sample 训练。

## 6. 缓解 / 趋势
- 厂商引入「轨迹分布器」：对每条 trace 做特征提取（dx 直方图 / 最大 dy / 0 抖动比例 / dt 熵）做 GBDT 打分。
- 合并设备指纹做联合判定。
- 越来越多关注 `trustedTypes` 与原生事件痕迹。
