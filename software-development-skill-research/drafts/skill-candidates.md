# 技能候选草案

## `software-development-core`

Description draft:

> General software engineering workflow for planning, reviewing, refactoring, testing, debugging, and safely delivering code changes. Use when Codex needs to make or review code changes across any stack, define implementation plans, assess quality risks, write tests, or prepare a change for commit/release.

Core sections:

- Clarify goal and constraints.
- Inspect existing code and local conventions.
- Choose implementation shape.
- Validate with risk-scaled tests.
- Review diff for correctness, maintainability, security, and rollback.

## `backend-development`

Description draft:

> Backend service and API development guidance covering REST/API design, database changes, caching, queues, auth, concurrency, observability, reliability, and production readiness. Use when working on server-side code, APIs, persistence, background jobs, or service integrations.

Core sections:

- API contract and compatibility.
- Data model and migrations.
- Reliability and idempotency.
- Observability and operations.
- Security and authorization.

## `frontend-development`

Description draft:

> Frontend engineering workflow for React/Vue/TypeScript applications, state management, routing, build performance, accessibility, browser performance, and frontend testing. Use when implementing web UI behavior, fixing frontend bugs, improving performance, or adding tests.

Core sections:

- State boundary selection.
- Component/data flow.
- Accessibility and keyboard behavior.
- Core Web Vitals and bundle impact.
- Component/E2E/visual testing.
